# Configuration

Here is some information about the configuration of the bot.

## Configuration File

The `config.js` file is located in the same folder as the bot executable. This file contains all the configuration options for the bot. You can find a list of all the configuration options below.

## Configuration Options

<Tabs>
<TabItem value="general" label="General">

### General Options

| Command             | Default | Description                                                                                                                                                                                                |
| ------------------- | ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `token`             | `-`     | Discord Bot token, You can obtain a bot token in the [Discord Developers Portal](https://discord.com/developers/applications/) make sure to use 1 bot per token, else you will get rate limited by Discord |
| `dbdAccountId`      | `-`     | Your account ID of your DBD account, more info about this [here](/docs/information#bot-activation)                                                                                                         |
| `guildId`           | `-`     | The Discord server ID where the bot will be used in                                                                                                                                                        |
| `updateIntervalSec` | `60`    | The amount of seconds between data updates by the bot                                                                                                                                                      |

</TabItem>
<TabItem value="presence" label="Presence">

### Presence Options

All options inside the `botPresence` config option.

| Command    | Default                                       | Description                                                                  |
| ---------- | --------------------------------------------- | ---------------------------------------------------------------------------- |
| `username` | `-`                                           | The username of the bot, keep empty if you don't want the name to be updated |
| `status`   | `Online`                                      | The status of the bot. (Options: Online, Idle, Dnd, Invisible )              |
| `activity` | `{totalplayers} players on {servers} servers` | The bot activity message that should be displayed                            |
| `type`     | `Watching`                                    | The status type. (Options: Playing, Streaming, Listening, Watching)          |

</TabItem>
<TabItem value="servers" label="Servers">

### Servers Options

:::info
You can list as many servers as you want, make sure to follow the format of the example config!
:::

All options inside the `servers` config option.

| Command     | Default           | Description                                                                                                  |
| ----------- | ----------------- | ------------------------------------------------------------------------------------------------------------ |
| `name`      | `Server1`         | The name of the server, this will be displayed in the bot activity message                                   |
| `ip`        | `123.123.123.123` | The IP of the server                                                                                         |
| `queryPort` | `12345`           | The query port of the server                                                                                 |
| `gameId`    | `arkse`           | The game ID of the server, you can find a list of all game IDs [here](https://www.npmjs.com/package/gamedig) |
| `joinLink`  | `-`               | The join link of the server, keep empty if you don't want the join link to be displayed                      |
| `info`      | `-`               | The info of the server, you can use this however you like. you can use it anywhere in the server fields.     |

</TabItem>
<TabItem value="counterPanels" label="counterPanels">

### CounterPanels Options

:::info
You can list as many counter panels as you want, make sure to follow the format of the example config!
:::

All options inside the `counterPanels` config option.

| Command      | Default   | Description                                                                                                          |
| ------------ | --------- | -------------------------------------------------------------------------------------------------------------------- |
| `uniqueId`   | `-`       | The unique ID of the counter panel, this is used to identify the counter panel. duplicated ids will result it issues |
| `channelId`  | `-`       | The channel ID where the counter panel should be displayed                                                           |
| `servers`    | `[]`      | The servers that should be displayed in the counter panel list them by their names from the `servers` list           |
| `formatting` | `options` | More information about formatting is listed below                                                                    |

### CounterPanels Formatting Options

All options inside the `counterPanels.formatting` config option. There are 2 types of fields in the counter panel, one is the server field and the other is the totals field. The server field is the field that displays the information about a specific server. The totals field is the field that displays the total amount of players and servers.

<Tabs>
<TabItem value="options" label="Server Field" default>

All options inside the `counterPanels.formatting.server` config option.

| Command   | Default | Description                                                                    |
| --------- | ------- | ------------------------------------------------------------------------------ |
| `title`   | `-`     | The title of the server field, this will be displayed above the server field   |
| `message` | `-`     | The message of the server field, this will be displayed below the server title |
| `inline`  | `true`  | Whether the server field should be displayed inline or not                     |

</TabItem>
<TabItem value="totals" label="Totals Field">

All options inside the `counterPanels.formatting.totals` config option.

| Command   | Default | Description                                                                    |
| --------- | ------- | ------------------------------------------------------------------------------ |
| `title`   | `-`     | The title of the totals field, this will be displayed above the totals field   |
| `message` | `-`     | The message of the totals field, this will be displayed below the totals title |
| `inline`  | `true`  | Whether the totals field should be displayed inline or not                     |
| `enabled` | `true`  | Whether the totals field should be displayed or not                            |

</TabItem>
</Tabs>

<details>
<summary>Available Placeholders for the server and totals fields</summary>

These placeholders can be used in the `server` and `totals` fields.

| Placeholder     | Description                                             |
| --------------- | ------------------------------------------------------- |
| `{statusEmoji}` | Displays a status emoji depending on the server status. |
| `{locked}`      | Displays a lock emoji if the server is locked.          |
| `{serverName}`  | Displays the name of the server.                        |
| `{map}`         | Displays the map of the server.                         |
| `{players}`     | Displays the amount of players on the server.           |
| `{maxPlayers}`  | Displays the max amount of players on the server.       |
| `{ping}`        | Displays the ping of the server.                        |
| `{playerList}`  | Displays a list of players on the server.               |
| `{disclaimer}`  | Only shows when a server is offline                     |

</details>
